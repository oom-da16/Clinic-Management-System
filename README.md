*1.)  Abstract*

Healthcare institutions face increasing challenges in managing patient records, scheduling 
appointments, and coordinating medical staff efficiently. The Clinic Management System is a 
Windows Forms desktop application developed in C# that addresses these challenges by 
providing a structured, user-friendly platform for clinic administrators and doctors. The system 
enables users to register patients, assign doctors, schedule and track appointments, and 
maintain medical records. Data is persisted using file handling, ensuring continuity between 
sessions. The application is built on core object-oriented programming principles including 
class hierarchies, inheritance, enumerations, polymorphism, and exception handling, making 
it both robust and extensible. The expected outcome is a fully functional clinic management 
tool that demonstrates real-world applicability of OOP concepts taught in CSE015. 

*2.) Problem Statement*


Small and mid-sized clinics often rely on manual, paper-based processes for managing 
patients and appointments. This leads to errors in records, scheduling conflicts, and inefficient 
use of doctor time. There is a need for a simple yet reliable digital solution that can be deployed 
on a standard Windows PC without requiring internet connectivity or expensive infrastructure.

*3.) Proposed Solution*


The Clinic Management System provides the following key features: 

• Patient Registration: Add, edit, search, and remove patient profiles.

• Doctor Management: Maintain a list of doctors with specialization details. 

• Appointment Scheduling: Book, confirm, or cancel appointments with status tracking. 

• Medical Records: Attach diagnoses and prescriptions to patient visits. 

• Data Persistence: Save and load all data using file streams (.txt files). 

• Input Validation: Exception handling ensures data integrity throughout the 
application.



*4.)Project Scope & Timeline*


• Patient class, registration form, file handling for patients. 

• Doctor class, doctor management form, inheritance from Person.

• Appointment class, scheduling form, enumerations for status. 

• Medical Record class, records form, array-of-objects management. 

• Main dashboard UI, navigation, exception handling throughout. 

• Integration, testing, class diagram, and documentation.
